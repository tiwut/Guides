Nextcloud on TrueNAS Scale

The ultimate guide to hosting your own private cloud using the power of ZFS and TrueNAS Scale's application ecosystem.

Introduction

☁️

Nextcloud is the industry standard for self-hosted file synchronization, sharing, and collaboration. Running it on TrueNAS Scale leverages the underlying Linux kernel (Debian-based) and the robustness of the ZFS file system.

This setup provides enterprise-grade data integrity, snapshots, and easy expandability, making it the perfect home or business server solution.

Prerequisites & Storage Setup

Before installing the application, you must prepare your storage to ensure data persists across updates and reboots. Do not rely on internal container storage.

1. Create Datasets

Navigate to Datasets in TrueNAS and create a parent dataset (e.g., apps) and children for Nextcloud:


# Recommended Dataset Structure
/mnt/tank/apps/nextcloud/data    # For your actual files
/mnt/tank/apps/nextcloud/config  # For Nextcloud configuration
/mnt/tank/apps/nextcloud/db      # For the PostgreSQL database

2. Permissions

Ensure the Apps user (usually UID 568 or 33 depending on the chart) has permission to write to these datasets. In TrueNAS Scale ACL editor:

  • Owner: apps (or www-data)
  • Group: apps (or www-data)
  • Mode: Read/Write/Execute

Installation Guide

TrueNAS Scale offers an "Apps" tab which utilizes Kubernetes (k3s) or Docker (Dragonfish and later). Follow these steps:

  1. Go to the Apps tab and click Discover Apps.
  2. Search for Nextcloud. You will likely see the Official iXsystems chart or community charts.
  3. Click Install.
Configuration Highlights:
  • Admin User: Set a strong username and password.
  • Storage: Switch from "IX Volume" to "Host Path" and map the datasets created in the Prerequisites section.
  • Network: Choose a port (default 9001) or configure Ingress if you have a reverse proxy (Traefik/Nginx Proxy Manager) set up.

Performance & Optimization

Nextcloud can be resource-intensive. Apply these settings to ensure a smooth experience.

Redis Caching

Enable the Redis sub-container within the App configuration. This drastically improves web interface speed and file locking.

PHP Memory Limit

Standard PHP limits are often too low for large file uploads. If available in your App settings, increase the memory limit:


# Environment Variable Example
PHP_MEMORY_LIMIT: 1024M
PHP_UPLOAD_LIMIT: 10G

Cron Jobs

Ensure Nextcloud is set to use Cron (system cron) rather than AJAX for background tasks. This is usually the default in the official TrueNAS app, but verify in Nextcloud settings under Administration > Basic Settings.

Maintenance & Security

Backup Strategy: RAID is not a backup.
  • ZFS Snapshots: Set up a periodic snapshot task in TrueNAS for your Nextcloud datasets (e.g., every hour). This allows you to roll back against ransomware or accidental deletion instantly.
  • External Access: Do not expose Nextcloud directly to the internet via port forwarding. Use a VPN (WireGuard/Tailscale) or a secure Reverse Proxy with Cloudflare Tunnel.
  • Updates: Always backup your config and database datasets before applying an App update in TrueNAS.
