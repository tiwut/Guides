Network Administration Guide

Welcome to the comprehensive guide on Computer Networking. This resource covers essential fundamentals, protocols, diagnostic tools, and security best practices for modern network administration.

Introduction

Computer Networking is the practice of transporting and exchanging data between nodes over a shared medium. Whether managing a Home Lab (LAN) or a corporate Wide Area Network (WAN), the goal remains the same: efficient, reliable, and secure communication.

This guide serves as a reference for understanding how data packets travel across the internet and how to troubleshoot connectivity issues effectively.

Key Concepts: Protocols & Models

  • The OSI Model: The 7-layer framework for data communication (Physical, Data Link, Network, Transport, Session, Presentation, Application).
  • TCP/IP: The practical suite of protocols used by the Internet (including IP addressing and TCP reliability).
  • DNS (Domain Name System): Translates human-readable domain names (tiwut.de) into machine-readable IP addresses.
  • DHCP: Automatically assigns IP addresses to devices on a network so they can communicate.
Recall: Networking issues are often solved by checking layers from the bottom up (Physical Layer first: Is the cable plugged in?).

Getting Started with CLI Tools

Every network administrator should master command-line interface tools for diagnostics. Here are the essentials:


# Check connectivity to a host (Latency & Packet Loss)
$ ping 8.8.8.8

# View Network Configuration (IP, Subnet, Gateway)
# Windows:
$ ipconfig /all
# Linux/macOS:
$ ip addr show

# Trace the path packets take to a destination
# Windows:
$ tracert google.com
# Linux/macOS:
$ traceroute google.com

# Check open ports and active connections
$ netstat -an

Use man [command] (Linux) or [command] /? (Windows) to view specific flags.

Best Practices & Security

Building a network is not just about connectivity; it is about keeping data safe. Follow these guidelines:

Caution: Never leave default credentials (admin/admin) on routers, switches, or IoT devices.
  • Network Segmentation: Use VLANs to separate Guest traffic, IoT devices, and sensitive Internal data.
  • Firewall Rules: Follow the principle of "Implicit Deny" ‚Äî block everything by default and only allow necessary traffic.
  • VPN Usage: Always use a VPN when accessing your home or business network remotely.
  • Redundancy: Implement backup power (UPS) and redundant links to prevent downtime during hardware failures.
