SQL (Structured Query Language) is the standard language for storing, manipulating, and retrieving data in relational databases. Think of it as the language you use to talk to the database to ask questions or give commands.

Note: SQL is technically NOT case-sensitive (select is the same as SELECT), but it is a best practice to write keywords in UPPERCASE for readability.

1. Anatomy of a Table

Data in SQL is stored in Tables. A table looks like a spreadsheet.

Example Table: users

id (Column) username (Column) email (Column) age (Column)
1 john_doe john@example.com 28
2 jane_smith jane@test.co 34
  • Table: The whole grid above (e.g., "users").
  • Column: A vertical category (e.g., "email"). Holds a specific type of data.
  • Row: A horizontal record. Represents one single item (e.g., User #1).

2. The SELECT Statement

The most common command. It allows you to select specific data from a table.

Select Everything

Use the asterisk (*) to grab all columns.

SELECT *
FROM users;

Select Specific Columns

Better for performance. Only grab what you need.

SELECT username, email
FROM users;

3. Filtering Data (WHERE)

üîç

Use the WHERE clause to filter results based on specific conditions.

SELECT *
FROM users
WHERE age > 18;

Common Operators

  • = : Equal to
  • <> or != : Not equal to
  • > / < : Greater/Less than
  • BETWEEN : Within a range
  • LIKE : Pattern matching (e.g., starts with 'A')

4. Modifying Data (CRUD)

Beyond reading data, you need to manage it.

INSERT (Create)

INSERT INTO users (username, age)
VALUES ('mike_jones', 22);

UPDATE (Modify)

Warning: Always use a WHERE clause with UPDATE. If you omit it, you will update every row in the table!
UPDATE users
SET age = 23
WHERE username = 'mike_jones';

DELETE (Remove)

DELETE FROM users
WHERE id = 1;
