Firewall Configuration for Better Security

A comprehensive guide to securing your network infrastructure. Learn how to configure rules, manage traffic flow, and harden your system against unauthorized access.

A Firewall acts as the first line of defense between your trusted internal network (or server) and untrusted external networks, such as the Internet.

Proper configuration is essential. An unconfigured firewall provides a false sense of security, while an overly restrictive one can disrupt legitimate services. This guide focuses on Linux-based firewalls (specifically UFW/iptables) as a standard for server security.

Core Policies & Concepts

  • Default Deny: The golden rule of security. Block all incoming traffic by default and only allow specific exceptions.
  • Ingress vs. Egress: Understanding the difference between incoming traffic (Ingress) and outgoing traffic (Egress).
  • Stateful Inspection: Modern firewalls track the state of active network connections, allowing return traffic automatically.
Principle of Least Privilege: Only open ports that are absolutely necessary for your services to function. Close everything else.

Basic Configuration (UFW)

For most Linux systems, UFW (Uncomplicated Firewall) is the standard tool. Here is how to set up a secure baseline:

1. Set Defaults and Allow SSH

Before enabling the firewall, you must allow SSH to prevent locking yourself out of the server.


# 1. Deny all incoming connections by default
sudo ufw default deny incoming

# 2. Allow outgoing connections (updates, curl, git)
sudo ufw default allow outgoing

# 3. Allow SSH (Port 22 is standard, adjust if you use a custom port)
sudo ufw allow ssh
# OR specific port: sudo ufw allow 2222/tcp

2. Enable the Firewall


# Activate the firewall settings
sudo ufw enable

# Check status and rules
sudo ufw status verbose

Advanced Rules & Best Practices

Once the baseline is set, optimize your configuration for better security and specific services.

Caution: When modifying rules on a remote server, always ensure your current SSH session remains active until you verify the new configuration works.
  • Rate Limiting: Protect SSH against brute-force attacks.
    sudo ufw limit ssh
  • Web Traffic: Allow standard HTTP and HTTPS traffic.
    sudo ufw allow 80/tcp (HTTP)
    sudo ufw allow 443/tcp (HTTPS)
  • Specific IP Whitelisting: Allow access to sensitive ports (like databases) only from specific trusted IPs (e.g., your VPN or office static IP).
    sudo ufw allow from 192.168.1.50 to any port 3306
  • Logging: Enable logging to audit blocked attempts.
    sudo ufw logging on
