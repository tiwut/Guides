Rewriting History

A clean commit history makes code reviews easier. Here are the tools to achieve it:

  • Interactive Rebase: Edit, squash, reorder, or drop commits.
  • Amending: Quickly modify the most recent commit.
  • Cherry-Picking: Applying specific commits from other branches.

Interactive Rebase

To clean up the last 3 commits before pushing:


# Opens your default editor with a list of commits
$ git rebase -i HEAD~3

# Commands available inside the editor:
# p, pick = use commit
# r, reword = use commit, but edit the commit message
# s, squash = use commit, but meld into previous commit
# d, drop = remove commit
Recall: Never rebase commits that you have already pushed to a shared repository (public history).

Recovery & Debugging

Git provides safety nets and automated debugging tools that often go unused.

The Reflog (Safety Net)

If you accidentally deleted a branch or reset to the wrong commit, the reflog is your savior. It tracks every movement of the HEAD pointer.


# View the history of your HEAD pointer
$ git reflog

# Reset to a specific point in time (e.g., before a bad rebase)
$ git reset --hard HEAD@{2}

Git Bisect (Automated Debugging)

Use binary search to find the exact commit that introduced a bug.


$ git bisect start
$ git bisect bad            # Current version is bad
$ git bisect good v1.0.0    # Last known good version

# Git will now check out the middle commit automatically.
# You test the code, then run 'git bisect good' or 'git bisect bad'.

Best Practices & Tips

Follow these guidelines to ensure a robust and collaborative workflow:

Caution: Always ensure your working directory is clean (or stashed) before starting a rebase or bisect operation.
  • Atomic Commits: Each commit should do one thing (e.g., "Fix login bug", not "Fix login bug and add footer").
  • Commit Messages: Use the imperative mood (e.g., "Add feature" not "Added feature").
  • Worktrees: Use git worktree to work on multiple branches simultaneously without switching contexts constantly.
  • Verification: Sign your commits with GPG/SSH keys for security.
