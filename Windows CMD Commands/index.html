Windows CMD Commands

A comprehensive reference guide for the Windows Command Prompt. Master the command-line interface to navigate, manage files, and troubleshoot your system efficiently.

Introduction

üíª

The Command Prompt (cmd.exe) is the default command-line interpreter for Windows. It allows users to interact with the operating system using text-based commands rather than a graphical user interface (GUI).

While PowerShell and Windows Terminal have introduced modern features, CMD remains essential for legacy batch scripting, system repair, and quick administrative tasks.

Fundamentals

  • Directory Structure: Windows uses backslashes (\) for paths (e.g., C:\Users\Admin).
  • Admin Privileges: Many system commands require running CMD as an Administrator.
  • Switches/Flags: Commands often accept options starting with a forward slash (e.g., /s, /f) to modify their behavior.
Recall: Commands are generally case-insensitive in Windows CMD, unlike Linux-based terminals.

Common Commands

Below is a list of the most frequently used commands for file management and networking:

File & System Navigation


:: Change Directory
cd C:\Windows        

:: List files in current directory
dir                  

:: Make a new directory
mkdir NewFolder      

:: Clear the screen
cls

Networking & Info


:: Check IP Configuration
ipconfig /all        

:: Test connectivity
ping google.com      

:: Display system information
systeminfo

Use help [command] or [command] /? to see specific options for any tool.

Best Practices & Tips

To ensure a smooth and safe experience when using the command line:

Caution: Commands like del (delete) and format are irreversible. Double-check your syntax before pressing Enter.
  • Tab Completion: Type the first few letters of a file or folder name and press Tab to auto-complete it.
  • Command History: Use the Up Arrow key to recall previously typed commands.
  • Abort: If a command is stuck or running too long, press Ctrl + C to stop it immediately.
  • Redirection: Use > to save command output to a text file (e.g., dir > list.txt).
