Automate your tasks with PowerShell & Bash

Your complete guide to system orchestration, batch processing, and workflow efficiency using modern shell scripting.

Introduction

Automation is the cornerstone of modern IT administration. By mastering PowerShell and Bash, you can eliminate repetitive manual tasks, ensure consistency across servers, and dramatically speed up your workflow.

Whether you are managing Windows Server environments or maintaining Linux infrastructure, script-based automation allows you to interact with the operating system at a deeper level than any GUI permits.

Core Differences

Key Concept: PowerShell passes Objects between commands, while Bash passes Text/Strings.
  • PowerShell (Windows/Cross-Platform):
    • Uses a Verb-Noun syntax (e.g., Get-Service).
    • Built on the .NET framework.
    • Ideal for managing Active Directory, Azure, and Windows configurations.
  • Bash (Linux/macOS):
    • Uses concise, command-based syntax (e.g., ls, grep).
    • Relies heavily on piping text streams and file manipulation.
    • Standard for cloud servers, CI/CD pipelines, and container management.

Practical Scripts

Below are real-world examples of how to automate common administrative tasks in both environments.

1. PowerShell: Monitoring High CPU Processes

This script retrieves the top 5 processes consuming the most CPU and exports the data to a CSV file.

# Get top 5 CPU processes
$HighCpu = Get-Process | Sort-Object CPU -Descending | Select-Object -First 5

# Display to console
$HighCpu | Format-Table -AutoSize

# Export to CSV for reporting
$HighCpu | Export-Csv -Path "C:\Logs\HighCpuUsage.csv" -NoTypeInformation

Write-Host "CPU Report generated successfully." -ForegroundColor Green

2. Bash: Automated System Backup

This script archives a specific directory, stamps it with the current date, and deletes backups older than 7 days.

#!/bin/bash

# Variables
SOURCE_DIR="/var/www/html"
BACKUP_DIR="/backups/web"
DATE=$(date +%Y-%m-%d)
ARCHIVE_NAME="backup-$DATE.tar.gz"

# Create backup
tar -czf $BACKUP_DIR/$ARCHIVE_NAME $SOURCE_DIR

# Remove backups older than 7 days
find $BACKUP_DIR -type f -name "*.tar.gz" -mtime +7 -exec rm {} \;

echo "Backup $ARCHIVE_NAME completed and old files cleaned."
Pro Tip: You can now run PowerShell on Linux (using PowerShell Core) to write cross-platform scripts that work on both Windows and Linux systems.
